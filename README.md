# Zahra — Fany's CV Assistant

A small voice + chat web app. Ask questions about Fany Louis-Mondésir's CV
and Zahra (her assistant) answers, by text or out loud, in English or French.

The front end is a **static site** (plain HTML/CSS + React from a CDN, no build
step). Answers are generated by a tiny **serverless function** (`/api/ask`) so the
app is **open to everyone** — visitors never enter an API key. The Anthropic key
lives only as a server environment variable.

## Files

| File | Purpose |
|------|---------|
| `index.html` | Entry point — loads fonts, React, Babel, the CV data and the app |
| `styles.css` | All styling |
| `app.jsx` | The React app (transformed in the browser by Babel) |
| `cv-data.js` | Fany's CV, exposed as `window.FANY_CV` — edit this to update the CV |
| `fany-avatar.png` | Zahra's portrait |
| `api/ask.js` | Serverless function that calls Anthropic with your server-side key |

## Deploy on Vercel (recommended — keeps the app open)

1. Push **the contents of this folder** to a GitHub repo (so `index.html` and the
   `api/` folder are at the repository root).
2. On <https://vercel.com> → **Add New… → Project** → import the repo.
   - Framework Preset: **Other**. Build Command: empty. Output: empty.
3. **Settings → Environment Variables**, add:
   - `ANTHROPIC_API_KEY` = your key from <https://console.anthropic.com/settings/keys>
4. **Deploy**. Your site is live at `https://<project>.vercel.app` and anyone can
   use it — no key prompt.

> The key is only on Vercel's servers, never sent to the browser. Set usage limits
> on the key and keep it out of the repository.

### If the deployment shows 404
Your files are probably inside a subfolder. Either move them to the repo root, or
set **Settings → Build & Deployment → Root Directory** to the folder that contains
`index.html` and `api/`.

## How answers are resolved

1. **Built-in AI** — used automatically inside the Anthropic/Claude preview
   environment (handy while editing).
2. **`/api/ask`** — on the deployed site, the front end POSTs the prompt to the
   serverless function, which calls Anthropic with the server key and returns the text.

## Run locally

Static preview only (the `/api` route needs the Vercel runtime):

```bash
python3 -m http.server 8000   # visit http://localhost:8000
```

To run the serverless function locally, install the Vercel CLI and run `vercel dev`
with `ANTHROPIC_API_KEY` set in a `.env` file.

## Notes

- A purely static host (e.g. GitHub Pages) cannot run `/api/ask`. Use Vercel (or
  another platform with serverless functions) for the open, key-free experience.
- Voice input uses the browser's Web Speech API (best in Chrome/Edge/Safari) and
  needs microphone permission + HTTPS (Vercel provides HTTPS).
- Spoken replies use the browser's speech synthesis; available voices depend on the device.
- To update the CV, edit `cv-data.js`.
